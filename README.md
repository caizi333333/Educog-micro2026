# "芯智育才"——基于AI大模型的微控制器课程智能化教学辅助平台

## 1. 项目概述

"芯智育才" 是一个以学生为中心，基于AI大模型技术，采用 Next.js、Genkit 等现代化技术栈构建的智能化、个性化微控制器课程教学辅助平台。它旨在通过深度融合人工智能于学习的每一个环节，为学生打造一个沉浸式、高效率、充满趣味的在线学习环境，助力未来卓越工程师的培养。

本项目不仅仅是一个知识展示的平台，更是一个能与学生实时互动、能洞察其学习状态、并能提供定制化辅导的 **“智能学习伙伴”**。它通过多模态的交互、即时的仿真反馈、数据驱动的学情分析和游戏化的激励机制，致力于解决传统教学模式中的核心痛点。

## 3. 核心功能与特色

### 3.1 交互式课程内容
*   **富媒体教材**：将课程内容组织为可交互的折叠面板，融合了清晰的文本讲解、可直接复制代码的示例，以及显式的“知识点-技能-思政”三维学习目标。
*   **知识图谱联动**：内容中的核心关键词被自动链接到知识图谱，学生可以一键跳转，在更宏观的视图中理解概念的上下文关系。

### 3.2 可视化知识图谱
*   **结构化知识网络**：将课程中所有抽象、零散的知识点构建成一个可交互、可视化的网络图谱，清晰地展示了它们之间的内在联系。
*   **探索式学习**：学生可以自由点击图谱中的任何节点，查看其详细定义，或按章节筛选，高亮显示与特定章节相关的知识地图。

### 3.3 AI 智能助教
*   **7x24小时在线答疑**：基于 Genkit 和大语言模型，能理解学生用自然语言提出的任何课程问题。
*   **多模态丰富解答**：AI 的回答不是单一的文本，而是包括核心文字解答、关键代码示例、辅助理解的图示，甚至推荐平台外的相关教学视频。
*   **引导式学习与透明度**：在回答问题的同时，AI还会提供相关的推荐实验和追问建议。当知识超出课程范围时，会明确告知信息来源于网络搜索。

### 3.4 沉浸式代码仿真 [核心亮点]
*   **AI驱动的仿真引擎**：学生可以编写或载入8051汇编代码。后端AI会对代码进行深度分析，模拟其在真实硬件上的执行过程。
*   **即时可视化反馈**：仿真的结果不再是冰冷的寄存器值，而是动态、可视化的前端呈现：LED闪烁、数码管显示、实时滚动的波形图等。
*   **故障注入与调试训练**：平台允许主动注入常见的硬件故障（如“引脚短路”、“晶振不起振”），AI会分析并解释该故障对代码行为的影响，锻炼学生的硬件调试和问题排查能力。

### 3.5 个性化学情分析
*   **现代化仪表盘**：为每个学生提供一个专属的学情分析仪表盘，直观展示总学分、知识掌握度、成就解锁进度等关键指标。
*   **多维度能力评估**：通过“知识原子掌握度热力图”和“布鲁姆认知能力雷达图”，从知识点和认知层级两个维度，精准地为学生“画像”。

### 3.6 自适应在线测评与智能推荐
*   **即时诊断与反馈**：提供“答后即反馈”的在线测试。完成一道题即可核对答案，并获得指向相关章节的复习链接。
*   **AI学习路径规划**：完成测试后，平台能根据学生的薄弱知识点，利用AI自动生成一份循序渐进的个性化学习计划，实现“因材施教”。

### 3.7 游戏化成就体系
*   **专属成就殿堂**：设立独立的成就页面，将学习过程中的关键行为转化为超过25个设计精美的徽章，并按类别进行展示，激励学生全面地与平台互动。

## 4. 技术实现方案

本项目采用业界前沿的技术栈，确保了平台的性能、可扩展性和开发效率。

*   **前端框架**: **Next.js (App Router)** & React 18。充分利用其服务器组件（RSC）、路由缓存和流式渲染能力，优化页面加载速度和用户体验。
*   **UI 与样式**: **ShadCN/UI** & **Tailwind CSS**。提供了一套美观、可访问性高且易于定制的组件库，保证了界面的统一性和开发效率。
*   **AI 功能实现**: **Genkit (Google AI)**。作为AI功能的核心，Genkit负责编排所有的大模型调用流程（Flows）。我们设计了多个独立的AI Flow，如`aiStudyAssistant`、`codeSimulationFlow`等，每个Flow都封装了完整的“数据检索(RAG)-模型调用-格式化输出”逻辑。其中，AI工具（Tools）的使用让大模型具备了调用外部函数（如查询章节）的能力，极大地增强了其智能性。
*   **数据可视化**: **Recharts**。用于绘制学情分析中的雷达图等，将复杂的数据以直观、美观的方式呈现。
*   **安全与合规**:
    *   **隐私保护**: 平台设有独立的隐私政策页面，明确告知数据处理方式。
    *   **用户匿名化**: 用户标识在后端处理时会通过 SHA-256 哈希算法结合Pepper进行脱敏，不在数据库中明文存储。
    *   **数据保留策略**: 学习日志数据仅保留180天，之后滚动删除。

## 5. 应用案例与效果

设想一位学生“小明”的学习之旅：
1.  **初识**：小明通过阅读“课程内容”开始了第一章的学习，并通过章节开头的学习目标明确了方向。
2.  **探索**：他对“中断”概念感到困惑，点击关键词跳转到“知识图谱”，直观地看到了中断与定时器、I/O端口的关系。
3.  **解惑**：他来到“智能助教”页面，提问“定时器中断和外部中断有什么区别？”。AI助教不仅给出了文字解释和代码示例，还推荐了一个讲解中断优先级的视频。
4.  **实践**：受到启发，小明进入“实验仿真”页面，载入了“定时器中断生成方波”的示例代码。他成功运行并看到了P1.0口输出的方波。接着，他尝试注入“晶振不起振”的故障，AI的仿真结果显示所有周期性输出的频率都变为0，并解释了原因，让他深刻理解了时钟信号的重要性。
5.  **检验**：小明参加了“在线测评”。在答错一道关于中断优先级的题目后，系统立即给出了正确答案和解析，并引导他复习第六章的内容。
6.  **提升**：完成测评后，系统生成了一份诊断报告，指出他在“中断系统”和“定时器”方面较为薄弱。他点击“获取个性化学习计划”，AI为他规划了一个包含“观看中断视频-复习第6章-完成一个中断相关的仿真实验”的学习路径。
7.  **激励**：在这个过程中，小明解锁了“仿真初体验”、“提问达人”、“故障排除者”等多个成就，并在“学情分析”页面看到了自己知识掌握度的提升。

这种“学-练-测-评-荐”的AI驱动的学习闭环，正是本项目致力于解决传统教学痛点，提升教学质量和学生学习体验的核心价值所在。

## 7. 将代码同步到本地或 GitHub

您可以随时使用 Git 将本项目的完整代码克隆到您的本地计算机，或将其推送到您自己的 GitHub 仓库中进行管理。

### 操作步骤

1.  **在 GitHub 上创建一个新的仓库**:
    *   访问 [GitHub](https://github.com/new) 并创建一个新的、空的仓库。请**不要**勾选“使用README、.gitignore或许可证进行初始化”选项。
    *   创建后，复制您的仓库URL，格式为 `https://github.com/YourUsername/YourRepoName.git`。

2.  **在“芯智育才”平台的终端中执行以下命令**:
    *   打开平台底部的终端（Terminal）。
    *   **初始化本地Git仓库**:
        ```bash
        git init -b main
        ```
    *   **将您的GitHub仓库添加为远程源** (请将下面的URL替换为您自己的仓库URL):
        ```bash
        git remote add origin https://github.com/YourUsername/YourRepoName.git
        ```
    *   **添加所有文件到暂存区**:
        ```bash
        git add .
        ```
    *   **提交您的初次代码**:
        ```bash
        git commit -m "Initial commit of EduCog Micro project"
        ```
    *   **将代码推送到GitHub**:
        ```bash
        git push -u origin main
        ```

3.  **在本地计算机上克隆仓库**:
    *   完成上述步骤后，您就可以在您的本地计算机上打开终端，使用 `git clone` 命令将项目代码下载到本地了：
        ```bash
        git clone https://github.com/YourUsername/YourRepoName.git
        ```

现在，您的代码就已经成功地保存在了 GitHub 和您的本地计算机上！
