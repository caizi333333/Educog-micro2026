# 侧边栏显示问题排查指南

## 问题描述
登录后看不到侧边栏（左侧导航菜单）

## 解决方案

### 方案1：清除缓存并重新登录
1. 访问 `/clear-auth` 页面清除所有登录状态
2. 重新访问网站，会自动跳转到欢迎页
3. 点击"立即登录"
4. 输入账号密码登录
5. 登录成功后会自动刷新页面并显示侧边栏

### 方案2：手动刷新页面
1. 登录成功后，按 F5 刷新页面
2. 或使用 Ctrl+Shift+R（Windows）/ Cmd+Shift+R（Mac）硬刷新

### 方案3：检查调试信息
1. 登录后访问 `/test-sidebar` 页面
2. 查看页面显示的调试信息：
   - 是否显示"已登录: 是"
   - 是否显示正确的用户名和角色
   - localStorage 中是否有 accessToken

## 技术原理

### 侧边栏显示条件
1. 不在公开页面（登录、注册、欢迎页等）
2. 用户已登录（`user` 对象存在）
3. 不在加载状态或已有用户信息

### 修复内容
1. **AuthContext 初始化逻辑优化**
   - 确保 refreshUser 完成后才设置 loading 为 false
   - 避免组件认为加载完成但用户数据还未准备好

2. **登录后页面刷新**
   - 使用 `window.location.href` 替代 `router.push`
   - 确保页面完全重新加载，触发 AuthContext 初始化

3. **AppLayout 显示逻辑**
   - 加载中且无用户时显示加载动画
   - 有用户信息时才显示侧边栏

## 测试步骤

1. **清除所有状态**
   ```
   访问: http://localhost:3000/clear-auth
   ```

2. **重新登录**
   - 使用测试账号：student / student123456
   - 登录成功后应该自动跳转并显示侧边栏

3. **验证侧边栏**
   - 左侧应显示导航菜单
   - 底部应显示用户头像和名称
   - 可以点击菜单项导航

## 如果仍有问题

1. **检查浏览器控制台**
   - 打开 F12 开发者工具
   - 查看 Console 是否有错误信息
   - 查看 Network 标签中 `/api/auth/me` 请求是否成功

2. **检查本地存储**
   - 在 Application 标签查看 Local Storage
   - 确认有 `accessToken` 和 `user` 数据

3. **尝试其他浏览器**
   - 使用隐私模式或其他浏览器测试
   - 排除浏览器缓存问题
